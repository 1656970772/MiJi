# 无限暖暖实时穿插技术资料卡

## 来源事实

- 来源文章：知乎专栏《无限暖暖—实时穿插技术》。
- 作者：`稻草人`。
- 所属专栏：`游戏&影视 — 逆向& 分享`。
- 页面显示编辑 / 更新时间：`2026-04-20 03:19 更新`。
- 页面主题链接文本包括：`模型穿插技术`、`计算几何`、`实时布料解算算法`、`Möller-Trumbore 射线交叉算法`、`顶点色哈希表`。
- 文章声明：`本文仅作为学习记录! 以下内容纯属虚构！`，并说明有相当一部分内容为 AI 整理，仅供参考。
- 文章主旨：围绕换装时避免模型穿插的技术做拆解，重点讨论内搭、外套、头发、帽子、配饰等部件在试衣间组合时的压扁、隐藏、刚体跟随和平滑联动。
- 外部官方佐证：Unreal Engine 官方访谈《Behind the Scenes of Infinity Nikki》在 2024-08-23 发布，其中 Infold Games 技术负责人谢伟波说明，《无限暖暖》为试衣间搭配开发了自研的基于几何的实时穿插处理算法，用于检测角色身体各部件之间的 interpenetration，并按规则变形或隐藏受影响区域；官方访谈还提到自定义规则、网格平滑、区域联动、预计算数据，以及头发帽子、外套内搭、裤装收腰等问题。
- 外部算法佐证：Möller 与 Trumbore 的论文《Fast, Minimum Storage Ray/Triangle Intersection》发表于 Journal of Graphics Tools 2(1), 21-28, 1997，DOI 为 `10.1080/10867651.1997.10487468`。该论文是常见的射线三角形相交算法来源之一。
- 外部 UE 佐证：Epic 的材质表达式文档说明 `VertexColor` 材质表达式可以使用网格的顶点色数据；这只佐证 UE 管线中可读取顶点色，不等于证明知乎文章中的全部规则实现。

## 核心问题

换装游戏的难点不是单件衣服，而是任意多层部件组合：内搭可能被外套覆盖，头发可能从帽子里穿出，项链和蝴蝶结可能与领口冲突，裤腰和上衣下摆可能互相插入。文章讨论的目标，是在玩家换装时快速判断这些冲突，并把结果转成可缓存、可渲染、可由美术规则修正的变形或隐藏数据。

## 一句话结论

我的判断：这篇文章描述的方案更像一套“预计算 + 换装时一次性几何检测 + 规则修正 + 顶点偏移/隐藏编码 + 渲染阶段应用”的防穿模生产管线，而不是每帧实时物理布料碰撞系统。

依据：知乎文章反复强调 T-pose、预烘焙投影点、换装时计算一次并缓存；Unreal 官方访谈也明确提到预计算系统用于降低运行时交叉检测成本。

## 算法拆解

### 1. 三层模型假设

文章把问题抽象为三层：裸模、内搭、外套。

- 裸模提供人体基准表面。
- 内搭是需要被压扁、推开或隐藏的对象。
- 外套保持外观优先，作为检测遮挡和穿插的外层边界。

我的判断：这个抽象适合换装系统，因为多数视觉问题来自“里面的东西不该露出来却露出来了”。它不追求布料真实受力，而是追求组合后视觉上合理。

### 2. 从内搭顶点投影到裸模

文章提出，对内搭顶点寻找裸模上的最近投影点，再建立从内搭顶点到该投影点的射线。若这条射线与外套三角形相交，就说明内搭在外套外侧或与外套发生了不期望的关系，可用相交距离估计需要压扁或隐藏的程度。

我的判断：这里的关键是把复杂的“内外层网格关系”转成大量局部射线检测。它比完整布料碰撞便宜，也更容易被缓存。

### 3. 投影点烘焙

文章提出，投影点由内搭与裸模在 T-pose 下的相对关系决定，因此可在 Editor 下预烘焙。存储方式不是直接保存世界坐标，而是保存裸模三角形的三个顶点 ID 和该投影点在三角形上的重心坐标。

我的判断：用三角形 ID + barycentric coordinates 存储投影点，比直接存三维坐标更稳。只要后续用对应三角形当前顶点位置重建，投影点就能跟随裸模形变或变换；但不能绝对写成“完全不受动作、位移、缩放影响”，因为重建精度仍依赖骨骼蒙皮、拓扑一致性和坐标空间处理。

### 4. 外套三角形粗筛

文章提出，不应让每条射线遍历外套所有三角形，而是先给外套建立 3D 空间网格。外套 AABB 被划分成多个小 AABB cell，每个 cell 记录穿过该空间格子的三角形 ID。

运行时先计算射线经过哪些 cell，再收集这些 cell 中的候选三角形，最后做精确射线三角相交。文章给出的 cell 一维索引形式为：

```cpp
idx = x + dimX * (y + dimY * z);
```

我的判断：这属于 uniform grid / spatial hash 风格的宽阶段加速。它适合换装场景，因为单件服装三角形分布相对稳定，可以在换装计算前后重复使用辅助数据。

### 5. Möller-Trumbore 精筛

文章把精确检测落到 Möller-Trumbore 射线三角形相交算法。该算法返回射线参数和三角形内的重心坐标式参数，常用于判断一条射线是否命中三角形。

外部佐证：Möller 与 Trumbore 1997 年论文的核心目标，就是用较少存储计算射线与三角形相交，避免预存三角形平面方程。

我的判断：文章把它用作“候选三角形精筛”是合理的。真正决定性能的不是这一个算法名，而是前面的候选集规模、射线数量、数据布局、SIMD 化、缓存和线程调度。

### 6. CPU 计算，GPU 应用

文章提出，穿插检测在 CPU 中完成，部分可用 SIMD 加速。换装时计算出内搭每个顶点需要偏移的位置和是否隐藏，并编码进网格信息。渲染阶段把这些数据传入 GPU：顶点着色器解码并应用顶点位置变化，然后再做 GPU 蒙皮；隐藏信息传入片元阶段做剔除。

我的判断：这条路径适合“换装时计算一次、后续持续显示”的需求。它把昂贵的几何关系计算放在换装瞬间，把每帧工作压缩成相对简单的顶点偏移和片元剔除。

## 美术规则层

文章最有工程价值的部分不是单个相交算法，而是规则层。

### 顶点色分组

文章提出，美术制作时绘制顶点色，并把顶点色映射到不同处理规则。例如同一颜色组可以代表一个硬质挂件、一段拉链、一组装饰或一个需要同隐同显的局部区域。

UE 外部佐证：Epic 文档说明材质可以读取网格顶点色数据；但知乎文章中的“顶点色到防穿模规则”属于作者对系统的拆解与推测，需要按文章来源理解。

我的判断：顶点色在这里不是单纯做材质混合，而是变成内容生产的标注通道。它让程序知道哪些点应被当成同一组处理，也让美术能用熟悉的编辑方式参与规则定义。

### 同隐同显

文章提出，一个颜色组内若某个顶点需要隐藏，则整个组隐藏。这样可以避免拉链、扣子、金属件只隐藏一半造成破碎感。

我的判断：这是换装防穿模里非常实用的视觉规则，因为用户通常更容易接受“整个配件被隐藏”，而不是看到一个硬质部件被局部切碎。

### 刚体平移和平滑

文章提出，对于硬质或半硬质部件，组内穿模顶点偏移可以求平均，再把平均偏移赋给全组顶点。这样部件整体移动，而不是每个顶点独立扭曲。

我的判断：这相当于把几何检测结果从“点级响应”提升为“部件级响应”。对蝴蝶结、吊坠、扣件、金属装饰尤其重要。

### 蔓延衰减与阈值隐藏

文章提出，深穿插点可带动周围顶点退让，利用相邻组找最深穿插点并向外扩散衰减；当穿插太深、角度太特殊、推开会变丑时，则直接打隐藏标记。

我的判断：这说明该系统承认“几何推开不是万能的”。好的防穿模系统必须允许在压扁、平滑、隐藏之间切换，而不是执着于每个点都被推到一个数学上合理的位置。

### 子网格配置表

文章提出，服装之间有配置表，可按子网格单独配置整体隐藏、忽略检测、充当透明空气等策略。

我的判断：这是生产管线真正能落地的关键。因为服装类别、剪裁、透明材质、硬质装饰和夸张造型太多，单一算法无法覆盖全部组合，必须给内容侧留规则入口。

## 落地风险

知乎评论区有一条很重要的现实补充：有评论提到游戏内仍可见不少穿模问题，作者回复说实际落地阻力很大，包括工具流搭建、制作标准、验收、美术理解、顶点色绘制、人员流动、知识储备和工作量堆积等。

我的判断：这类系统最难的地方不是 Möller-Trumbore 算法，也不是空间网格，而是把它变成稳定的内容生产流程。只要美术标注、验收标准、异常可视化、批量测试和人员培训跟不上，系统再聪明也会在海量服装组合里漏出问题。

## 适合与不适合

适合：

- 换装游戏的试衣间组合。
- 发型与帽子、内搭与外套、裤腰与上衣、配饰与领口等局部冲突处理。
- 换装瞬间允许短暂计算，之后反复显示同一组合的场景。
- 有大量服装资产、但能建立统一命名、分层、顶点色和配置规范的项目。

不适合：

- 战斗中每帧连续变化的真实多层布料自碰撞。
- 任意姿态下完全物理准确的布料层间碰撞。
- 没有内容制作标准、没有顶点色规范、没有调试可视化的资产堆叠项目。
- 所有衣服都需要透明、半透明、硬质、软质、动态摆动同时完美兼容的场景。

## 可迁移清单

- 给身体、内搭、外套建立明确 layer / priority / rule 配置。
- 在离线或 Editor 阶段烘焙“内搭顶点到裸模三角形”的投影信息。
- 为外层服装建立空间加速结构，减少射线需要检测的三角形数量。
- 对长且斜的射线做分段或更精细 traversal，避免射线 AABB 覆盖过多 cell。
- 为服装资产制定顶点色规则：同隐同显、刚体组、平滑组、忽略组、透明空气组。
- 把 pair 级计算结果缓存起来，换装时只在组合变化后重算。
- 做调试视图：显示射线、命中三角形、偏移方向、隐藏 mask、顶点色组、规则来源。
- 做批量验收：自动枚举高频服装组合，截图或输出穿插风险报告。
- 给 LOD 和低配平台准备降级策略：减少检测点、扩大隐藏阈值、优先处理第一视角可见区域。
- 建立异常处理表：某些特殊服装直接走手动配置，而不是强行算法化。

## 我的判断

这篇资料应归入：`虚幻 / 换装系统 / 计算几何 / 防穿模 / 服装规则管线 / 渲染数据编码`。

它的价值在于把“每套搭配手动修穿模”转成“几何检测 + 美术标注 + 规则配置 + 渲染编码”的生产思路。它的边界也很明显：文章自己声明有虚构和 AI 整理成分，因此不能把其中每个实现细节都当作《无限暖暖》官方真实源码；更稳妥的用法是把它当成一个可借鉴的防穿模系统设计草图，再用 Unreal 官方访谈和基础算法论文做外部锚点。
