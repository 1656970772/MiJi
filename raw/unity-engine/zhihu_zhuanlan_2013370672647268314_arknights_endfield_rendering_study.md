## 核心问题

这篇文章不是讨论完整引擎架构，而是逆向视角下的角色渲染拆解。重点是：一个写实比例更高、但仍要保留二次元可控性的角色，如何在延迟渲染主流程之外，用前向补绘、SDF 脸部光照、假次表面、拍扁法线、环境光整形、雨痕湿润层、风格化高光和后处理，把角色脸部稳定地推到美术目标上。

## 一句话结论

我的判断：文章最值得归档的不是某一个 shader trick，而是它呈现了一种“场景走延迟，角色用专门前向路径补控制权”的二游角色渲染思路。脸部效果尤其明显：它不信任纯物理法线和普通 NdotL，而是把 SDF、遮罩、LUT、假光源、拍扁法线和后处理组合起来，换取稳定、可调、少碎块的脸部观感。

依据：知乎文章明确提到整体为延迟渲染、角色最后再前向渲染一次；Unity 手册也说明延迟与 forward path 可以在管线中并存，半透明和不支持延迟的 shader 会走 forward。

## 渲染流程拆解

### 1. 整体管线

文章观察到整体为延迟渲染，角色会先完整写入 GBuffer，之后又通过前向渲染绘制一次，以满足特殊光照效果。作者推测角色写 GBuffer 可能服务于 GTAO 等后续屏幕空间效果。

我的判断：这种做法的逻辑是“让角色参与场景级屏幕空间效果，同时在最终外观上保留角色专属 shader 控制权”。代价是角色多绘制一次，收益是脸部、头发、描边、阴影和半透明细节不被通用延迟光照绑死。

### 2. GBuffer 与雨晴切换

文章提到 GBuffer 约分为 5 个，其中比较有意思的是单独计算潮湿粗糙度，并在延迟光照时根据遮罩切换雨天或晴天效果。

我的判断：这说明湿润效果不是单纯后处理贴一层水，而是进入了材质属性层，至少会影响粗糙度、法线、AO 或高光强度一类的光照输入。

### 3. 角色前向渲染

文章提到角色为前向渲染；不透明物体描边是不透明渲染，额发及相关描边属于半透明；额发阴影、睫毛阴影等是单独网格绘制。

我的判断：这是二游角色很常见但很工程化的取舍：把“脸是否好看”拆成多个独立可控图层，而不是指望一个统一材质模型自动解决所有问题。

## 脸部 Shader 重点

### 1. 超级 shader 与分支

文章称脸部 shader 很长，为减少变体，把很多计算塞进同一个超级 shader，通过分支区分材质或效果。作者也提醒，其中一些计算可能并非只属于脸，而是在其他材质效果上开启。

我的判断：这类超级 shader 适合大型项目集中维护角色效果，但风险是调试复杂、分支路径难读、平台差异更难排查。资料卡归档时应记录“这是作者按当前 shader 效果解读，不等同于官方设计文档”。

### 2. 基色与表情图集

文章描述的基色流程是：先采样基础色纹理，再把 UV 缩小映射到 4 宫格表情纹理里，根据当前帧索引采样表情贴图并混合；之后转到 sRGB，再采样 LUT 做校色，并得到暗部环境光颜色供后续插值。

我的判断：表情叠层用图集处理，适合把口型、表情、脸部微变化放在贴图层解决，避免每个表情都走复杂网格或材质变体。

### 3. 菲尼尔假次表面

文章认为脸部使用菲尼尔权重和遮罩，在基色边缘叠偏肉色 / 红色环境染色，模拟次表面散射层次。它还会根据相机前向量转入脸部局部空间，判断玩家是否正对角色脸。

我的判断：这不是物理 SSS，而是面向镜头和脸部边缘的美术假效果。它的价值在于让脸颊、侧脸和轮廓处更柔和，同时用遮罩限制到想要的区域。

### 4. 拍扁法线

文章提到将脸部法线的 Y 值拍扁，用于后续环境光采样时弱化鼻子、眼窝等真实网格结构带来的奇怪明暗；面部可使用圆形偏平法线，其他区域保留原法线结构。

我的判断：这是二次元脸部渲染的核心思路之一：脸部不完全按几何真实受光，而是先把法线“美术化”，再参与环境光或边缘光计算。

### 5. SH 环境光整形

文章称环境光来自三层精度的 SH 混合，并用拍扁法线采样。之后会转 HSV 调整：保持色相，按冷暖限制饱和度，暗部降低饱和度，同时用非线性方式让高饱和颜色更亮。

我的判断：这一步的目标不是“更物理”，而是把环境光压成更稳定、更亮、更少脏色的角色底色。对写实场景里的二游脸非常关键。

### 6. 雨痕湿润层

文章提到根据天气、水位高度、对象基础湿度等决定是否绘制雨痕；通过随时间滚动的水波纹贴图和易湿蒙版，构建 AO、法线、粗糙度、湿润遮罩与高光增强。

我的判断：这比单纯屏幕空间雨滴更深入，因为它进入了角色材质局部，并能让丝袜、衣服、脸等不同材质的湿润响应不同。落地关键是美术蒙版和参数范围。

### 7. SDF 法线与脸部漫反射

文章把脸部 SDF 纹理用于构造光照法线。做法大意是根据光源方向决定是否水平翻转 SDF UV，再把 SDF 值转成局部空间法线，最后转世界空间并与原法线按遮罩混合。

之后脸部漫反射走 SDF + Ramp：根据灯光方向相对脸部正前方的值，动态计算 smoothstep 范围，得到阴影过渡，再采样 ramp 纹理。

我的判断：SDF 不只用来“判亮暗边界”，还被转成法线参与更多光照计算，这是文章里最有迁移价值的一点。它解决了普通脸部网格法线在鼻子、眼窝、脖子接缝上产生碎光的问题。

### 8. 高光与假光源

文章提到高光部分会构造绑定在角色面朝向上的“虚拟太阳”，再混合真实光、虚拟光和视线方向，形成魔改版 half-vector；还会根据视线偏移采样自发光纹理，并利用雨痕法线采样 matcap 得到湿润高光。

我的判断：这是一套典型的“物理模型做底，美术假光源做方向约束”的高光系统。它不追求 GGX 的纯粹性，而是让脸部高光稳定出现在角色设计希望的位置。

### 9. 边缘光与颜色调整

文章提到边缘光包含多种模式：基于 Fresnel 和侧视权重的边缘光、背光时强制的边缘光、基于额外光源方向的侧光，以及亮部饱和度提升。边缘光还会被阴影、AO、上下方向遮罩等限制。

我的判断：这体现了“边缘光必须被约束”。没有遮罩和阴影限制的 rim light 会很快变成塑料感或穿帮光。

## 后处理

### 1. 单 pass 汇总

文章称后处理都压缩在一个 pass 中，包括锐化等；作者只重点说明效果明显的部分。

### 2. ARRI LogC3 + LUT

文章称色调映射使用 ARRI LogC3：先把线性颜色按公式转换到 LogC3，再采样 LUT 完成色调映射。

我的判断：这类做法的意义是把 HDR 线性颜色映射到一个更利于 LUT 调色的对数空间，让最终色彩更接近影视调色工作流，而不是直接用简单曲线压缩。

### 3. Bloom 与暗角

文章提到 Bloom 会从锐化结果中抽出高亮贡献，对 Bloom 纹理做曲线调整和 tint 后叠加；暗角基于屏幕中心距离、宽高比修正、falloff 和边缘 tint 混合。

我的判断：Bloom 和暗角在这里是整体观感的一部分：前者让高亮和湿润层更有空气感，后者收束画面边缘，引导视线回到角色。

## 可迁移清单

- 场景可走延迟主流程，角色关键材质保留前向补绘路径。
- 角色即使最终前向绘制，也可先写 GBuffer 参与 AO、深度、反射或其他屏幕空间效果。
- 脸部法线不要完全信任模型几何；可准备美术化法线、拍扁法线或 SDF 法线。
- 脸部阴影用 SDF + Ramp 时，必须处理左右光源、UV 镜像、脖子接缝和身体区域混合。
- 环境光进入角色前先做色相、饱和度、亮度整形，避免场景光把脸染脏。
- 高光系统可混合真实光、角色朝向假光、视线方向和 matcap，但要用遮罩限制。
- 雨痕 / 湿润不要只做屏幕空间后效；对角色可进入材质属性层，影响法线、粗糙度和高光。
- 后处理 LUT、Bloom、暗角要与角色 shader 联合调，而不是最后随手套一层。
- 所有逆向拆解都应附带“推测”标签，不能当作官方实现。

## 适合与不适合

适合：

- 二次元 3D 角色与写实场景融合。
- 角色脸部需要稳定、干净、少碎块的项目。
- 想做 PBR + NPR 混合角色渲染的 Unity / 自研渲染管线项目。
- 需要处理雨天、湿润、半透明头发和局部阴影网格的角色系统。

不适合：

- 追求完全物理一致的角色材质。
- 没有 TA 能维护超级 shader 和大量遮罩贴图的项目。
- 移动端预算极紧、不能接受角色多绘制或多贴图采样的项目。
- 缺少 RenderDoc / GPU 调试流程，只能靠肉眼调 shader 的团队。

## 我的判断

这篇资料应归入：`Unity / 游戏渲染 / PBR+NPR / 二游角色 / 脸部 shader / 后处理`，而不是归入虚幻目录。

它的价值是提供一张“高品质二游脸部渲染拆解图”：从 GBuffer、前向补绘，到 SDF 法线、Ramp 漫反射、假 SSS、SH 环境光整形、湿润层、假光源高光、边缘光、LogC3 + LUT。它的边界同样要明确：文章本身声明有虚构和 AI 整理成分，且作者是逆向学习视角，因此只能作为技术灵感和拆解线索，不能作为《终末地》官方技术白皮书引用。
